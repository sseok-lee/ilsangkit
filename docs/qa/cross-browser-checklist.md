# 크로스 브라우저 테스트 체크리스트

## 개요

일상킷 프로젝트의 모든 주요 브라우저에서 정상 동작을 보장하기 위한 체크리스트입니다.

### 테스트 대상 브라우저

| 브라우저 | 버전 | 플랫폼 | 우선순위 |
|---------|------|--------|---------|
| Chrome (Desktop) | Latest | Windows, macOS | P0 |
| Chrome (Mobile) | Latest | Android | P0 |
| Safari (Desktop) | Latest (17+) | macOS | P0 |
| Safari (iOS) | Latest (17+) | iOS | P0 |
| Samsung Internet | Latest | Android | P1 |
| Firefox | Latest | Windows, macOS | P1 |
| Edge | Latest | Windows | P2 |

### 테스트 뷰포트 크기

- **Mobile**: 320px ~ 767px (iPhone SE, iPhone 14, Galaxy S23)
- **Tablet**: 768px ~ 1023px (iPad, Galaxy Tab)
- **Desktop**: 1024px ~ 1920px (일반 모니터, 고해상도)

## 알려진 이슈 및 대응 방안

### Safari (iOS)

#### 이슈 1: 100vh 높이 문제
- **증상**: iOS Safari에서 `100vh` 사용 시 하단 주소창이 높이에 포함됨
- **대응**: CSS `dvh` 단위 사용 또는 JavaScript로 실제 viewport 계산
```css
.full-height {
  height: 100vh;
  height: 100dvh; /* 동적 viewport height */
}
```

#### 이슈 2: Backdrop Filter 성능
- **증상**: `backdrop-filter: blur()` 사용 시 성능 저하
- **대응**: 투명도 낮추기 또는 대체 스타일 적용

#### 이슈 3: Touch Event Delay
- **증상**: 버튼 클릭 시 300ms 지연
- **대응**: `touch-action: manipulation` CSS 속성 추가

### Samsung Internet

#### 이슈 1: CSS Grid Gap
- **증상**: 구버전에서 `gap` 속성 미지원
- **대응**: `margin` 대체 또는 최신 버전 권장

### Firefox

#### 이슈 1: Scrollbar Styling
- **증상**: `::-webkit-scrollbar` 동작 안 함
- **대응**: Firefox 전용 `scrollbar-width` 속성 사용

## 테스트 실행 방법

### 1. 로컬 개발 환경
```bash
# Frontend 개발 서버 시작
cd frontend
npm run dev
```

### 2. Playwright 자동 테스트 (기본 검증)
```bash
# 모든 브라우저 테스트
npm run test:e2e

# 특정 브라우저만
npm run test:e2e -- --project=chromium
npm run test:e2e -- --project=firefox
npm run test:e2e -- --project=webkit  # Safari 엔진
```

### 3. 실제 디바이스 테스트
- **Android**: Chrome DevTools Remote Debugging
- **iOS**: Safari Web Inspector (macOS 필요)

### 4. BrowserStack/LambdaTest (선택 사항)
- 클라우드 브라우저 테스트 플랫폼 활용
